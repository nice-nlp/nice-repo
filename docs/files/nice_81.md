# NICE 81期 | 从"看图说话"到"画图思考"，AI的认知跃迁

## 主题

从"看图说话"到"画图思考"，AI的认知跃迁

## 时间

2025.08.24 周日 21:30

## 内容

当大模型把静态“看图”升级为动态“用图”，视觉本身就成了推理的引擎。 告别“看图说话”，迎来“画图思考”。本次分享将围绕由香港科技大学、微软等联合撰写的最新 survey《Thinking with Images for Multimodal Reasoning: Foundations, Methods, and Future Frontiers》。这篇综述第一次系统梳理了多模态大模型如何从“被动看图像”（Thinking about Images）变成“主动用图像”（Thinking with Images），并给出“工具→程序→想象”的三阶段路线图。

本次分享围绕下几点展开： 
1 为什么必须“用图像思考”？ 
传统做法把图像一次性编码成向量后就只靠语言推理。遇到几何题、空间规划、机器人操作这类需要反复放大、画线、模拟的任务时，静态向量往往一次看错、步步皆错。“用图像思考”把视觉数据当成可以随时改写的草稿纸，让模型像人一样边想边画边验证。 
2 三阶段路线图 
• 阶段1：调用外部视觉工具——像搭积木一样组合检测、分割、OCR 等模块。
• 阶段2：自己写代码——生成 Python 脚本，现场画图、量尺寸、做实验，步骤透明可查。
• 阶段3：直接在脑子里“想象”——利用统一的生成模型产出中间图像，形成真正的视觉链式思考。
3 方法概览 
提示、微调、强化学习三条路线各自怎么落地；现有工作如何用它们教会模型“选工具、写代码、产图像”。
4 评测与应用 
汇总数学推理、STEM、GUI 交互、机器人、医疗、教育等七大场景的最新基准 —— “用图思考”到底能把性能推到哪。
5 开放问题 
计算开销、错误传播、安全可信、跨模态扩展——我们还需要解决哪些关键点？ 

当AI学会“在脑子里画图”，它才真正开始看懂世界。
